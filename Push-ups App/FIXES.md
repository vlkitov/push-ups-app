# 🔧 Исправления детектора для горизонтального положения

## Проблема
Детектор работал только когда телефон расположен вертикально. При размещении на полу (горизонтально) расстояние определялось неправильно.

## Причина
Использовалась только **Z-координата** из матрицы трансформации:
```swift
let distance = abs(transform.columns.3.z)  // ❌ Неправильно
```

При повороте устройства оси координат меняются, и Z больше не показывает расстояние до лица.

## Решение
Вычисляем **евклидово расстояние** (полное 3D расстояние от камеры до лица):

```swift
let position = simd_float3(
    transform.columns.3.x,
    transform.columns.3.y,
    transform.columns.3.z
)
let distance = simd_length(position)  // ✅ Правильно
```

Формула: `distance = √(x² + y² + z²)`

Это работает **независимо от ориентации устройства**.

## Как тестировать

1. Положите телефон на пол экраном вверх
2. Займите позицию для отжиманий лицом к камере
3. Запустите тренировку
4. Дождитесь калибровки (счетчик 0/30 → 30/30)
5. Начинайте отжиматься

## Отладочная информация

В процессе тренировки на экране будет отображаться:
- **Расстояние** - текущее расстояние до лица в метрах
- **Прогресс** - процент выполнения текущего отжимания (0-100%)
- **Фаза** - текущее состояние (вверху/внизу/переход)

## Типичные значения

При отжиманиях от пола с телефоном перед лицом:
- **Верхняя позиция**: ~0.35-0.45м от камеры
- **Нижняя позиция**: ~0.15-0.25м от камеры
- **Изменение**: ~0.20м (20 см)

Если значения сильно отличаются, можно настроить пороги в `PushUpDetector.swift`.

## Настройка порогов

Если детектор слишком чувствительный или наоборот:

```swift
// В PushUpDetector.swift

// Для более глубоких отжиманий (строже)
private let downThreshold: Float = 0.25  // 25см вместо 20см

// Для начинающих (мягче)
private let downThreshold: Float = 0.15  // 15см
private let upThreshold: Float = 0.10    // 10см
```
