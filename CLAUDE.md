# Push-ups App — Claude Context

## Обзор проекта

iOS-приложение для автоматического подсчёта отжиманий через TrueDepth (Face ID) камеру с помощью ARKit. Телефон кладётся экраном вверх на пол; ARKit отслеживает расстояние до лица и считает повторения.

**Требования:** iPhone X и новее (TrueDepth камера).

## Архитектура

```
ContentView.swift        — Главный экран с кнопкой "Начать тренировку"
WorkoutView.swift        — Экран тренировки (счётчик, фаза, прогресс-бар)
ARFaceTrackingView.swift — Обёртка ARKit-сессии (без видео, фоновый режим)
PushUpDetector.swift     — Основная логика детектирования (машина состояний, калибровка, сглаживание)
Push_ups_AppApp.swift    — Точка входа
```

### Поток данных

`ARFaceTrackingView` (ARSession delegate) → `PushUpDetector.processDistance(_:)` → `@Published` свойства → `WorkoutView` (SwiftUI)

## Ключевые параметры детектора (`PushUpDetector.swift`)

| Константа | Значение | Описание |
|---|---|---|
| `downThreshold` | 0.20м | Приближение от baseline для фиксации "вниз" |
| `upThreshold` | 0.08м | Расстояние от baseline для фиксации "вверх" (повторение засчитано) |
| `minimumRepInterval` | 0.5с | Антидребезг между повторениями |
| `smoothingWindowSize` | 5 | Окно скользящего среднего |
| Кадры калибровки | 30 | ~1 секунда при 30 FPS |

## Логика детектирования

- Используется **евклидово расстояние** `simd_length(position)` — работает при любой ориентации телефона
- **Машина состояний:** `.transition` → `.up` → `.down` → `.up` (повторение считается при переходе в UP)
- **Калибровка:** первые 30 кадров усредняются для установки `baselineDistance`
- `distanceChange = baseline - current` (положительное = лицо приближается / движение вниз)
